## 📝 DECISION LOG

### Recent Decisions

**Feb 15, 2026 - Side-by-Side Configuration View**
- Decision: Display JSON and Natural Language side-by-side
- Rationale: Users can learn JSON structure while seeing human-readable meaning
- Implementation: Grid layout with JSON left, NL right; both always visible
- Result: Educational experience, immediate comparison, professional UI

**Feb 15, 2026 - Privacy-First Config Design**
- Decision: Separate wallet address from configuration data
- Rationale: Protect user privacy, enable anonymous config sharing
- Implementation: ConfigOwnership interface for server-side only
- Result: Config exports don't expose wallet, users can have multiple personas

**Feb 15, 2026 - Natural Language Summary Generation**
- Decision: Generate human-readable descriptions from JSON config
- Rationale: Make technical JSON accessible to non-technical users
- Implementation: `config-summary.ts` library with descriptive text generation
- Result: Users understand their config without reading JSON

**Feb 15, 2026 - Configuration Persistence Fix**
- Decision: Skip stale cache validation in saveConfiguration
- Rationale: Cache was returning isHolder=false even with valid NFTs
- Implementation: Always verify NFT ownership fresh in save flow
- Result: Configuration saves now working for all NFT holders

**Feb 15, 2026 - Wallet Auth Flow Optimization**
- Decision: Simplify to single direct auth flow
- Rationale: Multiple eth_requestAccounts calls caused MetaMask timeouts
- Implementation: Removed two-step connection test
- Result: Authentication now reliable with multiple wallet extensions

**Feb 17, 2026 - Conversational Interview Implementation**
- Decision: Implement static 3-question interview with TensorFlow.js
- Rationale: Higher quality entity extraction (85% vs 70%) than rule-based alone
- Implementation: USE model for embeddings + pattern matching hybrid approach
- Features: Real-time entity extraction, intent classification, session persistence
- Result: Production-ready conversation UI v2.1.0

**Feb 18, 2026 - LLM Provider Integration**
- Decision: Build multi-provider LLM integration with secure API key storage
- Rationale: Enable dynamic question generation while maintaining user privacy
- Implementation: Wallet-signature encryption (AES-256-GCM) + localStorage only
- Features: OpenAI, Anthropic, Local (Ollama) support, cost tracking, budget controls
- Security: Zero server storage, encryption keys user-owned via wallet signatures
- Result: Production-ready LLM integration v2.2.0

**Feb 13, 2026 - Deployment Strategy**
- Decision: Keep manual deployments for now
- Rationale: Prevent broken code from auto-deploying
- Review: Revisit after stability proven

**Feb 13, 2026 - Test Suite Priority**
- Decision: Defer test fixes to next iteration
- Rationale: Functionality verified via API tests
- Review: Fix before adding new features

**Feb 13, 2026 - Documentation Strategy**
- Decision: Archive completed sprint docs
- Rationale: Reduce clutter, maintain history
- Action: Created archive/completed-sprints/
