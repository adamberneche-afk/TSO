#### 3. **Trust Score Algorithm (MEDIUM PRIORITY)** ✅ **DECIDED**

**Decision:** Multi-Factor with Accuracy Decay

**Formula:**
```
Trust Score = Base Score + Token Bonus + Accuracy Multiplier + Staking Bonus

Where:
- Base Score = Author (30pts) + Auditor (20pts) + Voucher (10pts)
- Token Bonus = up to 20% based on THINK holdings
- Accuracy Multiplier = 0.5x to 2.0x based on historical accuracy WITH decay
- Staking Bonus = up to 15% based on stake duration
```

**Accuracy Decay Mechanism:**
- Recent audits (last 30 days): 100% weight
- 30-90 days old: 75% weight
- 90-180 days old: 50% weight
- 180+ days old: 25% weight
- Never penalized for old accuracy, just reduced influence

**Why This Works:**
- ✅ Prevents auditors from resting on past reputation
- ✅ Rewards continued high-quality participation
- ✅ Reduces impact of outdated audit patterns
- ✅ Maintains incentive for ongoing engagement
- ✅ Balances experience with current competence

**Calculation Example:**
```
Auditor: Alice
- Base: 20 pts (auditor status)
- Token Bonus: 15% (holds 1000 THINK)
- Accuracy: 1.5x (90% accuracy, mostly recent)
- Staking: 10% bonus (staked for 6 months)

Score = 20 * 1.15 * 1.5 * 1.10 = 37.95 pts
```

**Implementation:**
- Store accuracy history with timestamps
- Recalculate decay daily via cron job
- Cache scores for performance (refresh every hour)
- Display "stale" indicator for auditors with decaying accuracy

**Timeline:** 1 week (can be added post-MVP)

#### 4. **Sandbox Enforcement (MEDIUM PRIORITY)** ✅ **DECIDED**

**Decision:** Option E - Tiered Security Approach

**Three-Tier System:**

**Tier 1: vm2 (Trusted Publishers - Trust Score > 0.80)**
- **Use case:** Established, highly-rated publishers
- **Technology:** vm2 with 5-second timeout
- **Permissions:** Relaxed (can access declared resources)
- **Overhead:** Minimal (~10ms startup)
- **Security:** Good for trusted sources

**Tier 2: Docker Containers (Standard Publishers - Trust Score 0.50-0.80)**
- **Use case:** New or moderately-rated publishers
- **Technology:** Docker with restricted capabilities
- **Permissions:** Containerized isolation
- **Overhead:** Medium (~500ms startup, 50MB memory)
- **Security:** Strong isolation, harder to escape

**Tier 3: Native OS Sandbox (Untrusted/Flagged - Trust Score < 0.50)**
- **Use case:** New publishers, flagged skills, high-risk permissions
- **Technology:** seccomp (Linux) / Seatbelt (macOS) / Windows Sandbox
- **Permissions:** Minimal (network-only, no filesystem)
- **Overhead:** Low (~50ms startup)
- **Security:** Maximum isolation

**Automatic Escalation Rules:**
- Critical risk permissions → Force Tier 2 or 3
- First-time publisher → Start at Tier 2, promote after 5 clean audits
- Violation detected → Demote to stricter tier
- Manual override option for users

**Benefits:**
- ✅ Performance optimization for trusted skills
- ✅ Strong security for untrusted sources
- ✅ Graduated trust system
- ✅ Resource efficiency
- ✅ Clear security boundaries

**Implementation:**
- Detect OS at runtime
- Configure sandbox based on trust score
- Monitor violations and auto-adjust tiers
- Allow user overrides in config

**Timeline:** 2 weeks (Tier 1 for MVP, add Tiers 2-3 post-launch)

- **Distribution:**

  - 30% to stakers (passive income)
  - 20% to platform treasury
  - 10% burned (deflationary)

**Web2 User Solution - Fiat On-Ramp:**

**The Problem:** Most users don't have crypto wallets or THINK tokens

**The Solution - "Gas Station" Pattern:**

1. **Accept USD Payment:**
   - Credit/Debit cards (Stripe)
   - PayPal
   - Apple Pay / Google Pay
   - Bank transfer for enterprise

2. **Auto-Conversion Behind the Scenes:**
   ```
   User pays $1.00 USD
   ↓
   Platform converts $1.00 → 100 THINK (at current price)
   ↓
   Platform pays 0.5 THINK fee from treasury
   ↓
   Transaction executes
   ↓
   User never sees crypto, but ecosystem benefits
   ```

3. **Implementation Options:**

   **Option A: Managed Wallets (Recommended)**
   - Platform creates custodial wallets for web2 users
   - User authenticates with email/password or OAuth
   - Platform manages keys securely
   - User can optionally withdraw to personal wallet
   - **Pros:** Seamless UX, familiar web2 experience
   - **Cons:** Platform custody risk (mitigated with audits)

   **Option B: Pre-funded Subscriptions**
   - User buys "installation credits" with USD
   - Credits = fixed amount of THINK locked at purchase price
   - Credits consumed per installation
   - **Pros:** Predictable costs, no volatility exposure
   - **Cons:** Less transparent, more complex accounting

   **Option C: Direct DEX Swaps**
   - USD payment → USDC → THINK swap on DEX
   - Fully on-chain, non-custodial
   - **Pros:** Transparent, decentralized
   - **Cons:** Requires wallet, gas fees, complex for beginners

**Recommended Implementation:**

**Hybrid Approach:**
- **Web3 Users:** Pay directly with THINK from their wallet (Option C)
- **Web2 Users:** Managed wallets with fiat on-ramp (Option A)

**User Journey Comparison:**

| Step | Web3 User | Web2 User |
|------|-----------|-----------|
| Install skill | `tais install weather-api` | `tais install weather-api` |
| Authentication | Sign with wallet | Sign in with Google/GitHub |
| Payment | Pay 0.5 THINK | Pay $0.50 USD |
| Behind scenes | Direct blockchain tx | Auto-convert USD→THINK |
| Receipt | On-chain tx hash | Email receipt |
| Optional | View on Etherscan | Withdraw to personal wallet |

**Technical Implementation:**

1. **Stripe Integration:**
   ```typescript
   // Backend API
   app.post('/create-installation-payment', async (req, res) => {
     const { skillHash, userId } = req.body;
     const feeInTHINK = calculateFee(skillHash);
     const feeInUSD = await convertTHINKtoUSD(feeInTHINK);
     
     const paymentIntent = await stripe.paymentIntents.create({
       amount: feeInUSD * 100, // cents
       currency: 'usd',
       metadata: { skillHash, userId, feeInTHINK }
     });
     
     res.json({ clientSecret: paymentIntent.client_secret });
   });
   ```

2. **Auto-Conversion Flow:**
   ```typescript
   // After successful USD payment
   async function processInstallation(userId, skillHash, usdAmount) {
     // 1. Get current THINK price from oracle
     const thinkPrice = await getTHINKPrice();
     
     // 2. Calculate THINK amount
     const thinkAmount = usdAmount / thinkPrice;
     
     // 3. Check if user has managed wallet
     let userWallet = await getManagedWallet(userId);
     if (!userWallet) {
       userWallet = await createManagedWallet(userId);
     }
     
     // 4. Fund wallet from treasury
     await transferFromTreasury(userWallet.address, thinkAmount);
     
     // 5. Execute installation with THINK fee
     await executeInstallation(userWallet, skillHash);
     
     // 6. Distribute fees per tokenomics
     await distributeFees(thinkAmount);
   }
   ```
